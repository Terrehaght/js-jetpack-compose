Quick Start

Get up and running in minutes with this step-by-step guide

1

Create Your HTML File

Create a basic HTML structure with an app container and include the framework scripts.

manifest.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Android Web App</title>
    <style>
        * { box-sizing: border-box; margin: 0; padding: 0; }
        #app { height: 100vh; display: flex; flex-direction: column; }
    </style>
</head>
<body>
    <!-- Main Application Container -->
    <div id="app"></div>

    <!-- Framework Scripts -->
    <script src="https://i-spani.web.app/androidlayouts.js"></script>
    <script src="AndroidFramework.js"></script>
    <script src="MainActivity.js"></script>
</body>
</html>
2

Create Your MainActivity

Create an Activity class extending the base Activity and implement lifecycle methods.

MainActivity.js
// Destructure the framework components
const { Activity, Intent, Application } = AndroidFramework;
const { Column, Row, TextView, MaterialButton } = AndroidLayouts;

class MainActivity extends Activity {
    onCreate(savedInstanceState) {
        super.onCreate(savedInstanceState);

        // Build your UI using Compose-like syntax
        const view = Column({
            width: 'match_parent',
            height: 'match_parent',
            gravity: 'center',
            gap: 16,
            padding: 24
        }, [
            TextView({
                text: 'Welcome to Android Web!',
                textSize: 24,
                textStyle: 'bold'
            }),
            MaterialButton({
                text: 'Click Me',
                onClick: () => this.toast('Hello!')
            })
        ]);

        this.setContentView(view);
    }
}

// Launch the application
Application.setupBackButton();
Application.launch(MainActivity);

Pro Tip

Use the toast() method inherited from Activity to show quick feedback messages to users.

Architecture Overview

Understanding the framework's core structure

The Android Framework for Web is structured around familiar Android concepts, making it easy for Android developers to build web applications:

Component Description Android Equivalent
Activity Single screen with lifecycle management android.app.Activity
Fragment Reusable UI component within an Activity androidx.fragment.Fragment
Intent Message object for navigation & data passing android.content.Intent
Bundle Key-value container for state/arguments android.os.Bundle
ActivityManager Manages activity stack and navigation ActivityManager/NavController

Activity

The fundamental building block representing a single screen

An Activity represents a single screen in your application. It manages its own UI and has a defined lifecycle that you can hook into.

Creating an Activity

Extend the base Activity class and override lifecycle methods as needed:

JavaScript
class ProfileActivity extends Activity {
    onCreate(savedInstanceState) {
        super.onCreate(savedInstanceState);

        // Get data from Intent
        const intent = this.getIntent();
        const userId = intent.getExtra('userId');

        // Set up your view
        this.setContentView(this.buildUI(userId));
    }

    onStart() {
        super.onStart();
        // Activity is becoming visible
    }

    onResume() {
        super.onResume();
        // Activity is now interactive
    }

    onPause() {
        super.onPause();
        // Another activity is taking focus
    }

    onBackPressed() {
        // Handle back button
        this.finish();
    }

    buildUI(userId) {
        return Column({ padding: 20 }, [
            TextView({ text: `Profile: ${userId}` })
        ]);
    }
}

Key Activity Methods

Method Description
setContentView(view) Sets the activity's root view (accepts HTMLElement or component)
findViewById(id) Finds an element by its ID within the activity
getIntent() Returns the Intent that started this activity
startActivity(intent) Launches a new activity with the given Intent
finish() Closes this activity and returns to the previous one
toast(message, duration) Shows a toast notification message
runOnUiThread(callback) Executes callback on the next animation frame

Lifecycle

Understanding Activity and Fragment lifecycle states

Activities and Fragments follow a predictable lifecycle. Understanding this lifecycle helps you manage resources efficiently and provide a smooth user experience.

Activity Lifecycle Flow

onCreate()

Initialize UI and components

onStart()

Activity becoming visible

onResume()

Activity is interactive

onPause()

Losing focus, save state

onStop()

No longer visible

onDestroy()

Activity being destroyed

State Preservation

Use onSaveInstanceState(outState) to save state before the activity is paused, and onRestoreInstanceState(savedInstanceState) to restore it.

Intent

Navigate between Activities and pass data

Intents are messaging objects used to request actions from other activities. They can carry data (extras) and control navigation behavior through flags.

Basic Navigation

JavaScript
// Create an intent targeting another activity
const intent = new Intent(DetailActivity);

// Add data to pass along
intent.putExtra('userId', 123);
intent.putExtra('username', 'John Doe');

// Start the activity
this.startActivity(intent);

Retrieving Intent Data

JavaScript
class DetailActivity extends Activity {
    onCreate(savedInstanceState) {
        super.onCreate(savedInstanceState);

        // Get the intent that started this activity
        const intent = this.getIntent();

        // Retrieve extras with default values
        const userId = intent.getExtra('userId', 0);
        const username = intent.getExtra('username', 'Guest');

        // Get all extras at once
        const allExtras = intent.getExtras();
    }
}

Intent Flags

Flags modify the behavior of navigation:

Flag Behavior
FLAG_ACTIVITY_CLEAR_TOP Clears all activities above the target in the stack
FLAG_ACTIVITY_SINGLE_TOP Reuses existing instance if already at top of stack
FLAG_ACTIVITY_NEW_TASK Starts activity as a new task
FLAG_ACTIVITY_CLEAR_TASK Clears entire activity stack before starting
Using Intent Flags
const intent = new Intent(MainActivity);

// Clear entire stack and start fresh
intent.addFlags(
    Intent.FLAG_ACTIVITY_CLEAR_TASK,
    Intent.FLAG_ACTIVITY_NEW_TASK
);

this.startActivity(intent);

Bundle

Store and retrieve key-value pairs for state management

Bundles are containers for key-value pairs, commonly used to pass arguments to Fragments or save instance state.

JavaScript
// Create a new bundle
const bundle = new Bundle();

// Store values
bundle.put('name', 'John Doe');
bundle.put('age', 25);
bundle.put('isActive', true);
bundle.put('settings', { theme: 'dark' });

// Retrieve values with defaults
const name = bundle.get('name', 'Unknown');
const score = bundle.get('score', 0);

// Check if key exists
if (bundle.has('name')) {
    // Key exists
}

// Remove a key
bundle.remove('age');

// Get all data as object
const allData = bundle.getData();

// Clear all data
bundle.clear();

Using Bundles with Fragments

JavaScript
// Create fragment with arguments
const fragment = new UserProfileFragment();

const args = new Bundle();
args.put('userId', 123);
args.put('showHeader', true);

fragment.setArguments(args);

// In the fragment, retrieve arguments
class UserProfileFragment extends Fragment {
    onCreateView(savedInstanceState) {
        const args = this.getArguments();
        const userId = args.get('userId');
        // Build UI with userId...
    }
}

Fragments

Modular, reusable UI components within Activities

Fragments represent portions of UI that can be added, removed, or replaced within an Activity. They have their own lifecycle that mirrors the parent Activity's lifecycle.

Creating a Fragment

JavaScript
class ItemListFragment extends Fragment {
    onCreateView(savedInstanceState) {
        // Build and return your view
        const view = Column({
            padding: 16,
            gap: 12,
            width: 'match_parent'
        }, [
            TextView({
                text: 'Item List Fragment',
                textSize: 20,
                textStyle: 'bold'
            }),
            Divider(),
            // Add more content...
        ]);

        return view;
    }

    onViewCreated(view, savedInstanceState) {
        super.onViewCreated(view, savedInstanceState);
        // Initialize view components after view is created
        this.requireActivity().toast('Fragment loaded!');
    }

    onResume() {
        super.onResume();
        // Fragment is now visible and interactive
    }
}

Fragment Transactions

Use FragmentManager to add, replace, or remove fragments:

JavaScript
// In your Activity
class ContainerActivity extends Activity {
    onCreate(savedInstanceState) {
        super.onCreate(savedInstanceState);

        // Create a container for fragments
        const layout = Column({ height: 'match_parent' }, []);
        const container = document.createElement('div');
        container.id = 'fragment-container';
        container.style.flex = '1';

        layout.getElement().appendChild(container);
        this.setContentView(layout);

        // Load initial fragment
        this.loadFragment();
    }

    loadFragment() {
        const fragment = new ItemListFragment();

        // Pass arguments
        const args = new Bundle();
        args.put('category', 'featured');
        fragment.setArguments(args);

        // Add fragment to container
        this.getSupportFragmentManager()
            .beginTransaction()
            .replace('fragment-container', fragment, 'item-list')
            .commit();
    }
}

Fragment Transaction Methods

Method Description
add(containerId, fragment, tag) Adds a fragment to the container
replace(containerId, fragment, tag) Replaces existing fragment in container
remove(fragment) Removes the specified fragment
hide(fragment) Hides the fragment without destroying
show(fragment) Shows a previously hidden fragment
addToBackStack(name) Adds transaction to back stack
commit() Commits the transaction

Navigation

Navigate between screens and handle back navigation

Starting Activities

JavaScript
// Simple navigation
const intent = new Intent(SettingsActivity);
this.startActivity(intent);

// Navigation with data
const intent = new Intent(ProductDetailActivity);
intent.putExtra('productId', 42);
intent.putExtra('productName', 'Widget Pro');
this.startActivity(intent);

// Navigate and clear history
const intent = new Intent(LoginActivity);
intent.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TASK);
this.startActivity(intent);

Starting Activity for Result

JavaScript
const REQUEST_CODE_SELECT = 1;

// Start activity expecting a result
const intent = new Intent(SelectionActivity);
this.startActivityForResult(intent, REQUEST_CODE_SELECT);

// Handle result
onActivityResult(requestCode, resultCode, data) {
    if (requestCode === REQUEST_CODE_SELECT) {
        if (resultCode === AndroidFramework.RESULT_OK) {
            // Handle successful result
            const selected = data.selectedItem;
        }
    }
}

// In SelectionActivity - setting result
selectItem(item) {
    this.setResult(AndroidFramework.RESULT_OK, { selectedItem: item });
    this.finish();
}

Back Button Handling

JavaScript
// Set up global back button handler
Application.setupBackButton();

// Override in Activity for custom behavior
class EditorActivity extends Activity {
    onBackPressed() {
        if (this.hasUnsavedChanges) {
            // Show confirmation dialog
            this.showSaveDialog();
        } else {
            // Default behavior - close activity
            ActivityManager.finishActivity();
        }
    }
}

Keyboard Support

The ESC key is automatically bound to trigger onBackPressed() when Application.setupBackButton() is called.

Activity Manager

Programmatically control the activity stack

The ActivityManager provides methods to control activities and the navigation stack.

JavaScript
const { ActivityManager } = AndroidFramework;

// Get current activity
const current = ActivityManager.getCurrentActivity();

// Get the activity stack
const stack = ActivityManager.getActivityStack();
console.log(`Stack depth: ${stack.length}`);

// Finish current activity
ActivityManager.finishActivity();

// Clear all activities (logout scenario)
ActivityManager.clearAllActivities();

// Start a new activity directly
const intent = new Intent(HomeActivity);
ActivityManager.startActivity(intent);
Method Description
startActivity(intent) Starts a new activity from an Intent
finishActivity() Finishes the current activity
getCurrentActivity() Returns the currently active Activity
getActivityStack() Returns an array of all activities in the stack
clearAllActivities() Destroys all activities in the stack

Layouts

Compose-like declarative layout system

The framework provides a Compose-like syntax for building layouts. Layouts are provided by the AndroidLayouts library.

Basic Layouts

JavaScript
const { Column, Row, FrameLayout, GridLayout, ScrollView } = AndroidLayouts;

// Column - vertical stack
const column = Column({
    width: 'match_parent',
    height: 'wrap_content',
    padding: 16,
    gap: 12,
    gravity: 'center',
    background: '#f5f5f5'
}, [
    /* children */
]);

// Row - horizontal stack
const row = Row({
    width: 'match_parent',
    gravity: 'space_between',
    gap: 8
}, [
    /* children */
]);

// ScrollView for scrollable content
const scrollable = ScrollView({
    width: 'match_parent',
    height: 'match_parent'
}, [
    Column({}, [ /* long content */ ])
]);

Layout Properties

Property Values Description
width 'match_parent', 'wrap_content', number Element width
height 'match_parent', 'wrap_content', number Element height
padding number or { all, top, right, bottom, left } Inner spacing
gap number Space between children
gravity 'start', 'center', 'end', 'space_between', 'space_around' Child alignment
background CSS color string Background color

Component Examples

JavaScript
// TextView
TextView({
    text: 'Hello World',
    textSize: 18,
    textStyle: 'bold',
    textColor: '#333'
})

// MaterialButton variants
MaterialButton({
    text: 'Primary Button',
    variant: 'filled',
    leadingIcon: 'bx bx-check',
    onClick: () => console.log('Clicked!')
})

MaterialButton({
    text: 'Outlined',
    variant: 'outlined'
})

// Material Input
MaterialInput({
    label: 'Email Address',
    placeholder: 'Enter your email',
    type: 'email',
    onInput: (e) => console.log(e.target.value)
})

// Dialogs (static methods)

MaterialDialog.confirm(
    'Confirm Action',
    'Are you sure?',
    {
        onConfirm: () => Snackbar.success('Confirmed!'),
        onCancel: () => Snackbar.show('Cancelled')
    }
);

// Snackbar notifications
Snackbar.show('Message sent');
Snackbar.success('Successfully saved!');
Snackbar.error('An error occurred');
Snackbar.show('Item deleted', {
    action: { text: 'UNDO', onClick: () => {} },
    duration: 5000
});

Important

Never use browser native alert(), confirm(), or prompt() functions. Always use MaterialDialog methods instead for a consistent Material Design experience.
